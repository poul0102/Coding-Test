## In-place 방문 처리
```
import java.util.*;

class Solution {
    public int solution(int[] cards) {
        // 각 그룹의 크기 저장 리스트
        List<Integer> group = new ArrayList<>();
        
        // 1-based -> 0-based
        for (int i = 0; i < cards.length; i++) {
            cards[i] -= 1;
        }
        
        // 모든 카드를 시작점으로 탐색
        for (int i = 0; i < cards.length; i++) {
            // 이미 방문한 카드 건너뛰기
            if (cards[i] == -1) continue;
            
            int count = 1;
            int cur = cards[i];
            cards[i] = -1;
            
            // 이미 방문한 카드가 나올 때까지 탐색
            while (cards[cur] != -1) {
                int next = cards[cur];
                cards[cur] = -1;
                cur = next;
                count++;
            }
            
            group.add(count);
        }
        
        if (group.size() < 2) return 0;
        
        Collections.sort(group, Collections.reverseOrder());
        
        return group.get(0) * group.get(1);
    }
}
```


## visited 배열 
```
class Solution {
    public int solution(int[] cards) {
        boolean[] visited = new boolean[cards.length];
        
        int first = 0;  // 가장 큰 그룹
        int second = 0; // 두 번째로 큰 그룹
        
        for (int i = 0; i < cards.length; i++) {
            if (visited[i]) continue;
            
            int count = 0;
            int cur = i;
            
            while (!visited[cur]) {
                visited[cur] = true;
                cur = cards[cur] - 1;   // 1-based -> 0-based
                count++;
            }
            
            // 가장 큰 두 그룹 크기 갱신
            if (count > first) {
                second = first;
                first = count;
            } else if (count > second) {
                second = count;
            }
        }
        
        return first * second;
    }
}
```

In-place 방문 처리 : 원본 배열 수정을 통한 방문 처리 / 리스트를 활용한 그룹 크기 저장, 정렬 후 상위 2개 선택  
visited 배열 : visited 배열을 통한 방문 처리 / 그룹 크기 저장 없이 탐색 중 최대 크기 갱신  
=> 방문 정보와 카드 정보를 분리하여 관리, 그룹 크기 저장 없이 최대 그룹 크기를 갱신하는 ' visited 배열'방식이 더 효율적
